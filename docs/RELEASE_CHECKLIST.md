# GitHub Release Checklist

Use this checklist before publishing the repository.

1. Choose and add a repository licence.
2. Keep `THIRD_PARTY_NOTICE.md` in the root directory.
3. Review `README.md` and confirm that the description matches the intended audience.
4. Check that `docs/DETAILED_USAGE_GUIDE.md` is up to date.
5. Confirm that the ABS database files are intended for distribution.
6. Confirm that no private notes, personal names, unpublished ideas or proprietary materials are included.
7. Run any available script checks.
8. Tag a release version.
9. Invite early users to open issues for missing fields, unclear modes and broken workflows.
