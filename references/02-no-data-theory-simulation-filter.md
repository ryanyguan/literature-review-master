# No-Data Theory and Simulation Filter

## Purpose

Some projects do not have real-world data, proprietary data, experimental data or direct access to field evidence. The literature review must therefore decide whether a gap can be addressed with theory, formal modelling, equilibrium analysis, proofs, numerical experiments, computational experiments or simulation.

## Required classification

Every idea validation or gap analysis under no-data constraints must classify the gap.

```text
Type 1. Theory-viable gap
Type 2. Theory plus simulation viable gap
Type 3. Algorithmic or computational gap
Type 4. Empirical gap that is not viable without data
Type 5. Conceptual review or synthesis gap
```

## Evidence substitutes

When no real data exist, viable evidence can include these.

```text
Formal model
Equilibrium characterisation
Existence and uniqueness proof
Comparative statics
Threshold or regime map
Counterexample to a common intuition
Algorithmic guarantee
Complexity result
Simulation-based robustness
Numerical experiment
Literature-calibrated parameter range
Policy or welfare comparison inside the model
```

Simulation is not empirical evidence unless it is linked to real observed data. Treat simulation as numerical evidence about the model.

## Forbidden language for no-data papers

Do not write these unless real data exist.

```text
we empirically show
the data reveal
our empirical results
we validate using field data
causal effect
identification strategy
```

Use precise alternatives.

```text
our analysis shows
the model predicts
the numerical experiments illustrate
the simulations quantify
the comparative statics reveal
the regime map shows
```

## Journal risk classification

For each target outlet, classify no-data feasibility.

```text
Strong, for journals that accept pure theory or theory plus numerical evidence.
Conditional, for journals that accept theory only when disciplinary contribution is very clear.
Weak, for outlets that usually expect empirical support, field evidence, behavioural data or real system validation.
```

Do not recommend an empirical framing when the project has no data. Recommend a theoretical, analytical, computational or simulation-supported framing instead.
