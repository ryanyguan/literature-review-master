# Framework Difference versus Cosmetic Difference Test

## Core question

Does the proposed idea change the analytical framework, or merely relabel an existing one?

## Test dimensions

Compare the current idea against the closest papers.

```text
Actors
Actions
Timing
Information
State variables
Uncertainty
Objective functions
Constraints
Equilibrium or solution concept
Feasible set
Main comparative static
Welfare or performance criterion
Numerical design
```

## Strong framework difference

A framework difference is strong when at least one of these changes occurs.

```text
A new strategic response changes equilibrium behaviour.
A constraint changes the feasible set and optimal policy.
A new information structure changes beliefs or incentives.
A dynamic state changes the result relative to a static model.
A stochastic structure changes risk, waiting, inventory, queueing, project timing or failure behaviour.
A method unlocks a previously intractable problem class.
A parameter creates a non-trivial threshold or regime switch.
```

## Cosmetic difference

A difference is cosmetic when the paper only changes labels.

```text
Manufacturer becomes platform.
Retailer becomes brand.
Customer becomes investor.
Delay becomes waiting cost without changing timing.
Demand function is renamed engagement.
Simulation parameter names change but the theorem does not.
```

## Required verdict

Classify the idea.

```text
Framework-distinct
Partially distinct but risky
Cosmetic and likely unpublishable at the target level
```

When partially distinct, specify the exact modelling change required to become defensible.
