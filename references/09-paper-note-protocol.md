# Paper Note Protocol

## Paper note structure

Each important paper should have a note with these sections.

```text
Bibliographic record
Relevance tier
Reading pass level
Research question
Model or method
Key assumptions
Equilibrium or solution concept if applicable
Main results
Evidence and proof audit
Limitations
Connection to the current project
Collision risk
Citable claims
Exact quotations if permitted and necessary
Writing craft observations
```

## Theory paper extraction

For theory papers, record these.

```text
Actors
Timing
Information
Actions
Payoffs or objectives
Constraints
Uncertainty
Equilibrium or optimality concept
Existence and uniqueness logic
Main propositions
Comparative statics
Boundary conditions
Numerical experiments
Extensions
```

## Empirical paper extraction

For empirical papers, record these even when the user's own project has no data.

```text
Data source
Sample
Identification strategy
Key variables
Main empirical regularity
What the empirical evidence can and cannot support
Whether the paper motivates a theoretical model
```

## Citable claim extraction

Do not cite a paper for a claim unless the paper actually supports that claim. Record exact support level.

```text
Directly supports
Partially supports
Background support only
Does not support
```
