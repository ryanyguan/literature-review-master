# Killer Paper Protocol

## Purpose

A killer paper is a paper that makes the current idea already solved, obviously incremental, incorrectly framed, or unpublishable at the intended level.

## Search levels

Tier 0 is direct collision search. Search the exact mechanism, model ingredients, solution concept and application setting.

Tier 1 is closest-neighbour search. Search papers with the same primitive structure but different application labels.

Tier 2 is method-neighbour search. Search papers using the same game, optimisation, algorithm, statistical model, Bayesian structure or simulation architecture.

Tier 3 is adjacent-field search. Search economics, marketing, finance, IS, OR, OM, engineering and computer science depending on the idea.

## Killer-paper reading depth

Abstract scanning is not enough for Tier 0 and Tier 1 papers. At minimum, extract the following.

```text
Model primitives
Timing
Information structure
Decision variables
Objective functions
Equilibrium or optimisation concept
Main theorem or proposition
Comparative statics
Numerical experiments
Claimed contribution
Boundary conditions
```

## Collision verdicts

Use these labels.

```text
Direct collision, meaning the same framework and same main result.
Framework collision, meaning the same model logic under another label.
Mechanism collision, meaning the same mechanism with a different setting.
Partial collision, meaning one core component is already solved.
Safe neighbour, meaning related but clearly distinguishable.
Useful foundation, meaning it should be cited and built upon.
```

## Final output

A killer-paper scan must include this table.

| Paper | Collision type | What overlaps | What remains distinct | Verdict impact |
|---|---|---|---|---|

Do not soften a direct collision. Recommend Drop-A or a major pivot.
