# Numerical and Table Verification

## Purpose

This protocol checks numerical claims in manuscripts, appendices, tables, figures, captions, simulation sections, algorithm comparisons and literature summaries.

It applies to external numbers from cited sources and internal numbers generated by the user's model, proof, code, table or simulation output.

## What to check

```text
Exact value
Rounding
Truncation
Unit
Currency
Scale
Sign
Percentage versus percentage point
Baseline denominator
Sample size
Parameter value
Seed count
Replication count
Confidence interval or credible interval
p-value or posterior probability
Runtime
Optimality gap
Approximation ratio
Benchmark comparison
Table and figure consistency
Text and table consistency
Appendix and main text consistency
```

## Academic rounding rule

Exact source values do not always have to be copied with every decimal. Rounding is acceptable only when all four conditions hold.

```text
The rounding is mathematically correct.
The rounding convention is consistent across the manuscript.
The rounded value does not change the interpretation.
The source or internal evidence remains traceable.
```

If rounding changes a conclusion, mark numerical error.

## High-risk numerical patterns

Flag these aggressively.

```text
96.555 percent reported as 97 percent when precision matters.
0.834 reported as 0.83 in one table and 0.8 elsewhere.
12 percent confused with 12 percentage points.
Increase reported as decrease.
Billion confused with million.
Parameter range changed between proof and simulation.
Simulation result described as robust after one baseline scenario.
A policy said to dominate when it only dominates in a subset of regimes.
```

## Simulation-specific checks

For pure theory and numerical simulation papers, check whether the manuscript overstates what the simulation proves.

Allowed.

```text
The numerical experiments illustrate the mechanism.
The simulations quantify the strength of the effect.
The regime map shows where the reversal occurs.
The benchmark comparison suggests that the policy performs well within the modelled environment.
```

Not allowed unless real data or external validation exist.

```text
The simulations empirically validate the mechanism.
The results prove real-world effectiveness.
The model demonstrates that firms will behave this way in practice.
```
