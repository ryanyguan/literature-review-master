# Related Work Writing

## Purpose

Turn literature analysis into manuscript prose without overclaiming.

## Required structure

A strong related work section should answer these questions.

```text
Which stream owns the core problem?
Which stream owns the method?
Which stream owns the application context?
Which papers are closest?
What exactly do they not do?
Why does that omission matter for this paper?
```

## Preferred paragraph logic

Use this sequence.

```text
Stream identity
What the stream has established
Closest papers
Precise limitation
How the present paper differs
Why the difference changes the conclusion
```

Avoid weak novelty language such as few studies have examined unless backed by a systematic search.

## British English and punctuation rule

For manuscript prose, use British English. Minimise colons and semicolons. Replace them with cleaner sentence structures when possible. Do not mechanically split every sentence. Keep paragraphs coherent and sufficiently developed.

Example.

```text
Weak: The literature has three limitations: it assumes fixed demand, ignores strategic delay, and treats service capacity as exogenous.
Better: The literature has three limitations. It assumes fixed demand, ignores strategic delay, and treats service capacity as exogenous.
```

Example.

```text
Weak: The model predicts higher profits; however, this occurs only when waiting costs are high.
Better: The model predicts higher profits. However, this occurs only when waiting costs are high.
```

## Citation discipline

Every literature claim must be mappable to citations. Use the claim-citation matrix before final prose.

## No-data caution

If the user has no real data, do not write as if empirical validation exists. Literature support can motivate a mechanism, but it does not validate the user's simulation.
